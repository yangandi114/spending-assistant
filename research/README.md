# Research Directory

**Course:** HKU 2025–2026 Sem 2, COMP1110 Group B12

This folder collects the research, requirement-analysis, and consolidated program-reference artefacts produced by the Research & Documentation track. It is the canonical reference for *what the team set out to build*, *what comparable tools already exist*, and *how the delivered code maps onto the original specification*.

## Contents

| File | Purpose |
|------|---------|
| `PROGRAM_DOCUMENTATION.md` | Single user-facing reference for the program — overview, architecture, installation, feature walkthrough, data formats, requirements-to-implementation map, and known issues. Consolidates `DOCUMENTATION.md`, the *Competitive Analysis & Requirements Specification* report, and the live source code. |
| `competitive_analysis_report_EN.docx` | The team's *Competitive Analysis & Requirements Specification* report (English). Five parts: project overview, requirements (use cases, FR/NFR, MoSCoW), competitor analysis (10 products across 8 design "camps"), comparative feature matrix, and conclusions / development recommendations. |
| `README.md` | This index. |

## Relationship to the parent project documents

The research folder complements — and where appropriate, supersedes — the following parent-level files:

- `../README.md` — repository quick-start and structure.
- `../DOCUMENTATION.md` — module-level API reference (functions, parameters, returns).
- `../PLAN.md` — original project plan, role assignment, and timeline.
- `../FEATURES.md` — flat feature list as seen by users.
- `../UI.md` — UI design notes.
- `../roles.md` — role definitions.

When `PROGRAM_DOCUMENTATION.md` and any parent file disagree, `PROGRAM_DOCUMENTATION.md` is the authoritative single source of truth for graders and external readers.
